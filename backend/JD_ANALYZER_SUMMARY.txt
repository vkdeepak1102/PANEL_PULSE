═══════════════════════════════════════════════════════════════════════════════

✅ JD ANALYZER API - LLM-Based Job Description Analysis

Analyzes Job Descriptions and extracts/classifies required skills:

  Key Skills:           Core technical requirements
  Mandatory Skills:     Non-negotiable prerequisites
  Good-to-Have Skills:  Optional enhancements

═══════════════════════════════════════════════════════════════════════════════

🎨 3 API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════════

1️⃣  POST /api/v1/jd/analyze
    └─ Analyze single Job Description
    └─ Returns: Key/Mandatory/Good-to-Have skills
    └─ Min JD length: 50 characters

2️⃣  POST /api/v1/jd/analyze-batch
    └─ Batch process multiple JDs
    └─ Max 10 JDs per request
    └─ Aggregated results with statistics

3️⃣  GET /api/v1/jd/health
    └─ Service health check
    └─ Verify configuration
    └─ GROQ API status

═══════════════════════════════════════════════════════════════════════════════

INSTRUCTION (I):
  You are a Senior Recruitment Manager.
  Extract and classify required skills from JD.
  Classify as: Key Skills, Mandatory Skills, Good-to-Have Skills

CONTEXT (C):
  10+ years experience in mentioned skills.
  Thoroughly analyze the JD.
  Base on explicit statements, not assumptions.

EXAMPLE (E):
  Key Skills: Playwright Typescript - core evaluation
  Mandatory: Automation experience - strict requirement
  Good-to-Have: SQL Knowledge - optional enhancement

OUTPUT RULES (O):
  Structured list format.
  No extra text, no assumptions.
  If JD too short: "JD is very short, need more info on the JD"

TONE (T):
  Professional, Structured, Precise
  Enterprise HR standard
